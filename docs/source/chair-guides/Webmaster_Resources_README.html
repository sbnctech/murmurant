New Technology Chair? Start Here

FIRST STEP: Make sure you have all credentials and access before anything else. Request access to the SBNCTech@gmail.com account from your predecessor - this is where password manager access and key accounts are managed.
How the Transition Works:
  • The departing Technology Chair leads the leadership transition execution at the end of their term
  • As the incoming chair, you should be involved and observe the process so you're prepared to do it yourself next time
  • New chairs are often eager to get started - completing the transition promptly is appreciated by everyone
  • Aim to complete the transition on or immediately after Feb 1 or Aug 1 (the first day of each new term)

Request These From Your Predecessor

Before your predecessor's term ends, get:
  • Access to SBNCTech@gmail.com account (this is critical - do this first!)
  • Password manager access (1Password/LastPass) with all service credentials
  • Wild Apricot admin login credentials
  • Server SSH credentials (mail.sbnewcomers.org)
  • Leadership Transition Portal login
  • GitHub repository access (added to sbnctech organization)
  • cPanel login for email forwarding
  • 30-minute handoff call to walk through systems
  • List of any ongoing issues or projects
Information about each board/committee position:
  • New officer's full name (as they want it displayed)
  • New officer's personal email address
Your First Big Task: After getting credentials, your most important job is installing the new leadership team. See HOW_TO_Install_New_Leadership.html in the HOW_TO_guides folder.

GitHub Repository

Technical documentation and code live in a GitHub repository. This is where you'll find detailed guides, troubleshooting info, and system documentation.

Repository: https://github.com/sbnctech/wa-dev

How to Navigate the Repository (For Non-Technical Users)

What is GitHub? GitHub is like Google Drive for code and documentation. You don't need to download anything - you can read all documents directly in your web browser.
Step-by-step to find documents:
  1. Go to github.com/sbnctech/wa-dev
  2. You'll see a list of files and folders - this is the repository contents
  3. Click any file ending in .md to read it (like ONBOARDING.md)
  4. The README.md file displays automatically at the bottom - this is your starting point
  5. Use the search bar at the top to find specific topics

What Are .md Files?

.md files are "Markdown" files - a simple text format that can display with nice formatting (headers, bold, lists, links). They're the standard for documentation on GitHub.

How to read .md files:

  • On GitHub: Just click the file - GitHub automatically renders it with formatting
  • On your Mac: Use MacDown (free app) to view .md files with formatting
  • In any text editor: You can open .md files in TextEdit or any editor - you'll see the raw text with simple markup like *bold* and # headers

Learn more: Markdown Guide - Getting Started

Key documents to read first:
  1. README.md - Overview and links to everything else
  2. ONBOARDING.md - Complete new tech chair guide
  3. CREDENTIALS.md - All account credentials and rotation procedures
  4. RUNBOOK.md - Troubleshooting guide
Tip: You don't need to understand the code files (ending in .py). Focus on the .md files - those are the documentation written for you.
